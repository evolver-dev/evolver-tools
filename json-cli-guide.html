Quick Start: Your First JSON Command

Feed any JSON file or pipe into evtool json-pretty and get beautifully formatted output:

$ echo '{"name":"evolver","users":[{"id":1,"role":"admin"},{"id":2,"role":"user"}],"active":true}' | evtool json-pretty
{
  "name": "evolver",
  "users": [
    {
      "id": 1,
      "role": "admin"
    },
    {
      "id": 2,
      "role": "user"
    }
  ],
  "active": true
}

That's all it takes. No jq to install. No python3 -m json.tool to remember. Just evtool json-pretty.

1. Format & Validate JSON

Everyday JSON formatting tasks — from pretty-printing to minification to validation.

Pretty-Print a File

$ evtool json-pretty data.json
{
  "name": "My App",
  "version": "2.0",
  "dependencies": [
    { "name": "click", "version": "8.0" }
  ]
}

Minify (Single-Line)

$ evtool json-pretty --minify data.json
{"name":"My App","version":"2.0","dependencies":[{"name":"click","version":"8.0"}]}

Validate Only

$ evtool json-pretty --validate broken.json
Invalid: line 3 column 5 — Expected ',' or '}'
$ echo $?
1
Pro tip: Use --validate in CI/CD pipelines to catch malformed JSON before it breaks your build.

Custom Indentation

$ evtool json-pretty --indent=4 config.json
{
    "key": "value"
}

2. Explore JSON Structure

When you encounter an unfamiliar JSON file, start by scanning its keys and structure.

List Top-Level Keys

$ evtool json-keys api-response.json
status
data
meta
error

Inspect Deep Structure

$ evtool json-keys config.json --depth 3 --flatten
app.name
app.version
app.database.host
app.database.port
logging.level
logging.file

For API responses with deeply nested objects, --flatten gives you the complete dotted path map of every key — invaluable for documentation and debugging.

3. Query & Extract Data

Two tools for extracting specific values from JSON: json-path (simple dot notation) and jsonql (advanced filtering).

Dot-Notation Queries (json-path)

Extract values by path — no learning curve:

$ evtool json-path response.json --path=data.users[*].name
[
  { "path": "$.data.users[0].name", "value": "Alice" },
  { "path": "$.data.users[1].name", "value": "Bob" }
]
$ evtool json-path response.json --path=meta.total
[
  { "path": "$.meta.total", "value": 42 }
]

Advanced Filtering (jsonql)

For SQL-like filtering and transformations:

$ echo '[{"name":"Alice","age":30},{"name":"Bob","age":25},{"name":"Charlie","age":35}]' | evtool jsonql -c ".[] | select(.age > 25)"
{"name":"Alice","age":30}
{"name":"Charlie","age":35}
$ evtool jsonql data.json -c "group_by(.category) | to_entries | map({(.key): length})"
{"tech": 3, "science": 2, "arts": 1}
Compared to jq: jsonql uses simpler, more intuitive syntax. No cryptic .[] | {key: .key, val: .val} — just .[] | select(.field > value).

4. Compare & Diff JSON

Find exactly what changed between two JSON files — useful for API regression testing, config drift detection, and code review.

$ cat a.json
{"name": "App", "version": 1, "features": ["a", "b"]}

$ cat b.json
{"name": "App", "version": 2, "features": ["a", "b", "c"]}

$ evtool json-diff a.json b.json
2 difference(s) found:

version: changed
  - 1
  + 2

features: array item added at [2]
  + "c"

json-diff walks every nested object and array, reporting structural diffs down to any depth. Zero dependencies — pure Python stdlib.

Merge JSON Files

$ evtool json_merge base.json override.json
{
  "name": "App",
  "version": 2,
  "features": ["a", "b"]
}
$ evtool json_merge base.json override.json --strategy deep
(deep-merges nested objects instead of replacing)

5. Transform JSON

Sort Keys Alphabetically

$ echo '{"z":1,"a":2,"m":3}' | evtool json-sort
{
  "a": 2,
  "m": 3,
  "z": 1
}

Flatten Nested JSON to Key=Value

$ echo '{"user":{"name":"Alice","address":{"city":"NYC","zip":"10001"}}}' | evtool json-flatten
user.name=Alice
user.address.city=NYC
user.address.zip=10001

Flattened output is perfect for .env files, environment variable injection, or flat configuration loading.

Apply JSON Patch (RFC 6902)

$ cat patch.json
[
  {"op": "replace", "path": "/version", "value": "3.0"},
  {"op": "add", "path": "/author", "value": "Evolver"}
]
$ evtool json-patch data.json --patch patch.json

7. Validate Against JSON Schema

Ensure your JSON data conforms to a schema before using it in production:

$ evtool json-schema-validate data.json schema.json
Valid ✓

$ evtool json-schema-validate bad-data.json schema.json
Invalid: /users/0 — 'name' is required
         /users/1/age — expected integer, got string

Uses the jsonschema library when available, or falls back to a pure-Python structural check with zero dependencies.

8. Real-World Workflows

API Response Pipeline

$ curl -s https://api.example.com/users | \
  evtool json-pretty --validate | \
  evtool json-path --path=data[*].email | \
  evtool json-to-csv > users.csv

Config Drift Detection

$ evtool json-diff production.json staging.json | head -20
6 difference(s) found:

database.host: changed
  - prod-db.example.com
  + staging-db.example.com

features: array changed at [3]
  - "experimental"

CI/CD JSON Validation

# In your CI pipeline:
$ evtool json-pretty --validate manifest.json && evtool json-schema-validate manifest.json schema.json
Valid ✓

Log Analysis

$ cat server.log | grep '^{' | evtool json-pretty --validate | evtool jsonql -c "select(.level == 'ERROR')"

9. Why Zero Dependencies Matters

Every JSON tool in evolver-tools uses only Python's standard library:

  • No npx/node_modules — pure Python, no JavaScript toolchain
  • No jq compilation — jq requires a C compiler or a binary download
  • No npm install — one pip install, 2 seconds, done
  • Works offline — perfect for air-gapped environments and CI
  • Cross-platform — same command on Linux, macOS, Windows/WSL
  • No version conflicts — your project's pandas/tensorflow won't break
