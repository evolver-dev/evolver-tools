Data & Analysis

1
Quick-Check a CSV Without Pandas
You got a CSV file. What's in it? What are the columns? Any bad data? You don't have pandas or Excel.
csv data
# Schema, preview, and stats in three commands
evtool csv-schema data.csv evtool csv-head data.csv evtool csv-stats data.csv
CSV Schema: data.csv Columns: 7 Rows: 1,234 Types: name(str), age(int), city(str), salary(float), joined(date), active(bool) Column Count Unique Nulls Mean Min Max ───────────────────────────────────────────────────────────────────── name 1,234 1,198 0 - - - age 1,234 62 2 34.2 18 72 salary 1,232 412 2 ¥82,450 ¥3K ¥450K city 1,234 165 0 - - -
Why this works: csv-schema infers types from data (no config), csv-head shows the first rows, csv-stats gives descriptive statistics. All pure Python stdlib — works in air-gapped or minimal environments.
2
Filter & Plot CSV Data in a Pipeline
You need to filter a CSV by a condition, then visualize the distribution — without loading a notebook or R.
csv chart
# Filter rows where age > 30, then plot a histogram of salaries
evtool csv-filter data.csv --column age --min 30 | evtool csv-chart --column salary
Filtered: 712 rows (age >= 30) Salary Distribution ────────────────────────────────────────────────────── 0–50K ████████████████████████████ 312 50K–100K ██████████████████████ 258 100K–150K ███████████ 132 150K–200K ██████ 65 200K–250K ██ 28 250K+ █ 17
Why this works: Pipes between tools are zero-cost — csv-filter outputs valid CSV to stdout, csv-chart reads it. No temp files needed. This is the Unix philosophy applied to data work.
3
Log Analysis: Find the Top Error Patterns
You have a 100MB server log. You need to find the most frequent error patterns, fast.
text devops
# Extract ERROR lines, normalize timestamps, count & sort
grep ERROR server.log \ | sed 's/\[.*\] //' \ | evtool text-stats --lines \ | evtool csv-sort --column count --reverse
Error Pattern Count ─────────────────────────────────────────────────────────── Connection timeout to database 1,247 Rate limit exceeded for API endpoint /v2/orders 892 Disk quota exceeded on /data partition 445 Invalid authentication token 312 Unrecognized field 'metadata' in request body 198
Why this works: text-stats counts line frequencies (like `sort | uniq -c` but with more options). csv-sort can sort the output. No awk, no Perl, no separate installs.
4
JSON Data: Validate, Pretty-Print, and Transform
You got a messy JSON API response. You need to validate it, make it readable, and convert it to CSV.
json text
# One pipeline: validate → pretty-print → convert to CSV
curl -s https://api.example.com/users \ | evtool json-pretty \ | evtool json-to-csv --flatten > users.csv
# Or just pretty-print with validation
cat messy.json | evtool json-pretty --sort-keys
{ "status": "ok", "users": [ { "id": 1, "name": "Alice Chen", "email": "alice@example.com" } ] }
Why this works: json-pretty validates AND pretty-prints (with optional key sorting). json-to-csv handles nested objects via --flatten. No jq, no Python one-liners to memorize.

System & DevOps

5
Server Health Snapshot in One Command
You SSH into a server and want a quick health overview — OS, disk, memory, CPU, uptime — in a single glance.
system devops
evtool system-info && evtool disk-usage /
System Information ──────────────────────────────────────── OS: Linux 6.6.114 x86_64 Platform: Ubuntu 22.04 LTS Python: 3.11.15 Hostname: prod-web-01 Uptime: 47 days, 3h 22m Disk Usage — / Size Type Path ────────── ──── ───────────────── 234.2G dir /var/log 18.5G dir /usr 4.2G dir /home 1.8G dir /etc 980.0M dir /tmp
Why this works: system-info collects OS, kernel, Python, and host info from stdlib (platform, os, socket). disk-usage wraps os.statvfs. No htop, no btm, no procps required.
6
Find & Kill a Stuck Process by Port
Port 8080 is in use. You have no idea which process. You need to kill it.
system devops
evtool process-kill --port 8080
Found process on port 8080: PID: 14329 Name: node User: deploy Memory: 84.2MB Kill process 14329 (node)? [y/N] y ✓ Process 14329 terminated
Why this works: process-kill uses /proc and /proc/net/tcp (Linux) or lsof (macOS). Interactive confirmation prevents accidents. Pure stdlib — no fuser, no lsof dependency.
7
Cron Expression Humanizer
You inherited a crontab with `0 5 15 * 5` and nobody remembers what it does.
devops utility
evtool cron-pretty '0 5 15 * 5'
0 5 15 * 5 → At 5:00 AM, on day 15 of the month, and on Friday
# Validate an entire crontab
evtool cron-check /etc/crontab
Why this works: cron-pretty parses the 5-field cron format and renders it in natural language. cron-check validates the entire file. Saves you from crontab.guru lookups.
8
SSL Certificate Expiry Check
Your monitoring says a cert is expiring. You need to check the actual expiry date for multiple domains — fast.
network devops
evtool cert-check example.com evtool cert-check api.example.com evtool cert-info example.com
Certificate: example.com:443 Subject: CN = example.com Issuer: CN = R3, O = Let's Encrypt Valid: 2026-03-15 → 2026-06-13 Days left: 12 ⚠️ SANs: example.com, www.example.com, api.example.com
Why this works: cert-check connects to the TLS endpoint using Python's ssl module, extracts the cert chain, and parses validity dates. No openssl CLI needed.

Network & Diagnostics

9
Full DNS Investigation
A domain stopped resolving. Is it DNS? Which records are affected? What's the TTL?
network
# Query A, AAAA, MX, and NS records
evtool dns-lookup example.com evtool dns-lookup example.com --type MX evtool dns-lookup example.com --type NS
A 93.184.216.34 (TTL: 3600) AAAA 2606:2800:220::1 (TTL: 3600) MX mail.example.com (priority: 10, TTL: 3600) NS a.iana-servers.net NS b.iana-servers.net
Why this works: Pure-python DNS resolution using socket.getaddrinfo for A/AAAA and a built-in DNS client for MX/NS. No dig, no nslookup, no host command required.
10
Find Where an IP Is (Geolocation)
You see a suspicious IP in your logs. Where is it connecting from?
network
evtool geo-ip 8.8.8.8 evtool ip-location 8.8.8.8
IP Address: 8.8.8.8 Country: United States City: Mountain View ISP: Google LLC Coordinates: 37.4056, -122.0775 Timezone: America/Los_Angeles
Why this works: geo-ip uses ip-api.com (free tier, no API key). ip-location uses a fallback source. Both output clean text, no JSON parsing by hand.

12
Verify a Downloaded File's Integrity
You downloaded a large file. The site says the SHA256 is abc123... You want to verify without extra tools.
crypto
# Compute hash and compare
evtool hash-file ubuntu-24.04.iso --sha256
SHA256(ubuntu-24.04.iso) = a1b2c3d4e5f6... Expected: a1b2c3d4e5f6... Match: ✓ VERIFIED
# Or check a file against a known hash file
evtool hash-check ubuntu-24.04.iso SHA256SUMS
Why this works: hash-file computes MD5/SHA1/SHA256/SHA512 using Python's hashlib (C implementation, fast). hash-check reads a standard SHA256SUMS file and verifies all entries. No shasum, no md5sum.
